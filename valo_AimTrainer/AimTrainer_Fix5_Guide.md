# valo_AimTrainer fix5 適用手順書(実測キャリブレーション版)

fix5 = ユーザー提供の実機録画(VALORANT / UE5とも1080p・60fps)を光学解析した**実測値**でパラメータを確定した版。

## 1. 実測結果サマリ

解析方法: 各フレーム間の画面移動量を位相相関で抽出し速度カーブ化。全試行を50%クロスで整列平均。時間分解能は60fps録画のため±17ms(整列平均によりさらに向上)。

| 項目 | VALORANT実測 | UE5(fix4)実測/理論 | fix5設定 → 理論値 |
|---|---|---|---|
| 加速 20-80% | **106ms** (n=11) | 40ms(速すぎ) | a=3000 → 106ms |
| 加速 10-90% | 149ms | 53ms | → 145ms |
| 解放停止 20-80% | **43ms** (n=8) | 40ms | D=7500 → 43ms |
| 逆キー減速 20-80% | **62ms** (n=3) | 24ms(効きすぎ) | 摩擦4.5 → 約60ms |
| しゃがみ 沈み/立ち | **311±8 / 300±0ms**、速度カーブのピーク/平均比1.2 | 250ms・比1.5 | T=0.30・指数1.3 |
| ジャンプ滞空 | **756±8ms** (n=3) | 667±17ms(実測。理論665msと一致=fix4ビルド確認) | v=370, g=1.0 → 755ms |

低信頼データ(参考): AD切り替えの直接計測(399/431ms)は録画の視差が小さくノイズ過大の可能性。フィット値からの予測は約250〜300ms。W/Sキーは前後移動のため光学計測不可(A/D値を採用。VALORANTは前後横とも同速のため妥当)。

## 2. 適用手順

1. zipをプロジェクト直下に展開(ソース2ファイルのみ、Config変更なし)。
2. ビルド(エディタ終了→VS、またはCtrl+Alt+F11。ヘッダ変更ありのためフルビルド推奨)。
3. **最重要: BP_AimTrainerCharacter の値リセット**
   BPを開き、`AimTrainer|Move` `AimTrainer|Jump` `AimTrainer|Crouch` の各プロパティに黄色い「↺」が出ていたら**すべてクリックして既定値に戻す**(BPの旧値がfix5の実測値を上書きするため)。
   新規プロパティ `CrouchEaseExponent`(1.3)が増えています。

## 3. 動作確認(at.MoveDebug 1 での期待値)

| 計測 | fix5期待値 |
|---|---|
| 0 → 95%最高速 | 約171ms |
| キー解放 → 停止 | 約71ms |
| 逆キー → 停止 | 約105ms |

- ジャンプが以前より「高く・長く」感じられれば正常(滞空665→755ms)
- しゃがみはわずかにゆっくり(250→300ms)かつ直線寄りの動きになる
- 逆キーでの停止が解放より遅くなった(=実測どおり。counter-strafeの利得ほぼなし)

## 4. 数値を再調整したい場合

実測に不満がある場合の対応式(BPで変更): 加速時間T(20-80%)にしたい → MoveAcceleration = 324/T[s]。解放停止T(20-80%) → MoveBrakingDeceleration = 324/T[s]。しゃがみ → CrouchTransitionTime直接指定。滞空T → JumpZVelocity = 490×T[s](GravityScale=1.0時)。
