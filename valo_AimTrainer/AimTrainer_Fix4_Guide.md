# valo_AimTrainer fix4 適用・動作確認手順書

fix4 = VALORANTとの差分分析に基づく再調整(移動応答・しゃがみ補間方式) + 実測ツール追加

## 1. 適用手順

1. zip をプロジェクト直下に展開して上書き(**今回はソース2ファイルのみ。Config変更なし**):
   - `Source/valo_AimTrainer/AimTrainerCharacter.h`
   - `Source/valo_AimTrainer/AimTrainerCharacter.cpp`
2. ビルド: エディタを閉じてVSで `Development Editor | Win64`、またはエディタ内 **Ctrl+Alt+F11**(Live Coding)。
   ※ヘッダのプロパティ構成が変わっているため、Live Codingで反映されない場合はエディタを閉じてフルビルドしてください。
3. **重要: BP_AimTrainerCharacter の値リセット確認**
   BPで `AimTrainer|Move` のプロパティを一度でも編集していた場合、**BPの値がC++の新既定値(8192/8192/0)を上書き**します。
   BPを開き、MoveAcceleration / MoveBrakingDeceleration / MoveBrakingFriction の右に黄色い「↺(既定値に戻す)」矢印が出ていたらクリックして既定値へ戻してください。
   また、廃止した `CrouchCameraInterpSpeed` が残留表示される場合はBPを再保存すれば消えます。

エディタ側のアセット追加作業はありません。

## 2. 実測ツール(at.MoveDebug)の使い方

VALORANTとの比較を「体感」ではなく「ミリ秒」で行うためのツールです。

1. PIE中にコンソールを開く(`@` または `半角/全角` 付近のキー)。
2. `at.MoveDebug 1` と入力(戻すときは `0`)。
3. 画面左上に以下が表示されます:
   - 常時: 現在速度(uu/s と最高速比%)
   - **0 → 95%最高速**: 停止状態からキーを押して最高速の95%に達するまでのms(緑)
   - **キー解放 → 停止**: 走行中にキーを離してから速度5以下になるまでのms(黄)
   - **逆キー → 停止**: 走行中に逆キーを押してから速度が反転点を通過するまでのms(橙)

### fix4既定値での理論値(実測がこの近辺なら正常)

| 計測 | 理論値 |
|---|---|
| 0→95%最高速 | 約63ms (540×0.95÷8192) |
| キー解放→停止 | 約66ms (540÷8192) |
| 逆キー→停止 | 約40〜55ms (制動+旋回摩擦) |

### VALORANT側の計測方法(比較用)

1. VALORANTの射撃場で、キー入力が映る形(キーボード表示ソフト or 音)で**できるだけ高fps(120〜240fps)の画面録画**を撮る。
2. 動画編集ソフトやフレーム送りで「キーを離したフレーム → 画面の流れが止まるフレーム」の枚数を数え、`枚数 ÷ 録画fps × 1000` でmsを算出。
3. 同じ操作を本プロジェクトで行い、at.MoveDebug の表示と比較する。

### 差が出たときの調整対応表(BP_AimTrainerCharacter の Details)

| 感じた差 | 動かすプロパティ | 方向 |
|---|---|---|
| 出だしがもたつく / 速すぎる | MoveAcceleration | 上げる / 下げる |
| 離した後に滑りすぎる / 止まりが硬すぎる | MoveBrakingDeceleration | 上げる / 下げる |
| 停止の終わり際だけ粘る感じが欲しい | MoveBrakingFriction | 0→2〜4 に上げる |
| A⇔D切り返しがぬるい / 鋭すぎる | MoveGroundFriction | 上げる / 下げる |
| しゃがみが速すぎ / 遅すぎ | CrouchTransitionTime | 下げる / 上げる(0.20〜0.30) |

## 3. 動作確認チェックリスト

- [ ] `at.MoveDebug 1` で 0→95% が約60〜70ms、解放→停止が約60〜70ms、逆キー→停止が約40〜55ms
- [ ] キー解放の停止が「短い一定の滑り→ピタッ」と直線的に止まる(以前の「最初ガクッ、最後だらだら」がない)
- [ ] A⇔D切り返しがキビキビ反転する
- [ ] しゃがみ/立ちのカメラが**毎回同じ時間(約0.25秒)**でスッと完了する。連打しても破綻しない
- [ ] 低天井でキーを離しても頭上が空くまで立たない(従来どおり)
- [ ] マウス: 同DPI・同感度値でVALORANTと振り向き距離が一致(fps上限を60⇔240に変えても一致)
- [ ] 既存機能(視点・WASD・ジャンプ・しゃがみ速度低下)がすべて動作
- [ ] 赤字エラーなし / Output Log に `[AimTrainer] IMC を登録しました`

## 4. fix4での既知の割り切り(今後の課題)

- カプセル(当たり判定)の高さ変化は従来どおり**即時**です(UE標準)。VALORANTはヒットボックスも時間をかけて変形すると推測されますが、CMCのカスタム移動が必要になるため今回は見送り(カメラのみ時間遷移)。
- ジャンプ(高さ/重力)はfix4の対象外(fix3値を維持)。
